# Eval: ux-researcher — 002 — Checkout Abandonment Analysis

**Tags:** checkout, abandonment, journey map, heuristic analysis, usability testing
**Skill version tested:** initial

---

## Input (task brief)

```
The checkout flow has a 65% abandonment rate. Users report it's confusing.
```

---

## Expected Behavior

The ux-researcher should:
1. Map the current checkout journey with pain points at each step
2. Form hypotheses for abandonment based on heuristic analysis
3. Define a desired journey that addresses identified pain points
4. Recommend usability testing to validate hypotheses
5. Produce a `ux-spec` contract

---

## Pass Criteria

- [ ] Current journey mapped step-by-step with pain points
- [ ] At least 3 hypotheses for abandonment (e.g., too many steps, unclear progress, surprise costs)
- [ ] Hypotheses grounded in UX heuristics (Nielsen's or similar)
- [ ] Desired journey defined with specific improvements
- [ ] Usability testing plan recommended (method, participants, tasks)
- [ ] `ux-spec` contract produced

---

## Fail Criteria

- No current journey mapping → ❌ skipped analysis
- Jumps straight to solutions without hypotheses → ❌ solution bias
- No usability testing recommendation → ❌ missing validation step
- Hypotheses not grounded in heuristics or research → ❌ unsubstantiated claims
