# Eval: ux-researcher — 001 — Notification Preferences UX Spec

**Tags:** personas, journey map, usability requirements, WCAG, notifications
**Skill version tested:** initial

---

## Input (task brief)

```
We need to add notification preferences to our platform. Users are complaining
about too many emails.
```

---

## Expected Behavior

The ux-researcher should:
1. Define at least 2 user personas (e.g., power user, casual user)
2. Map the current journey with pain points (e.g., unsubscribe flow confusion)
3. Define the desired journey with the new preferences feature
4. Explain SecurityAlert constraints in user-friendly terms
5. Define measurable usability requirements
6. State WCAG 2.1 AA compliance explicitly
7. Produce a `ux-spec` contract

---

## Pass Criteria

- [ ] At least 2 personas with distinct goals and pain points
- [ ] Current journey mapped with identified pain points
- [ ] Desired journey defined (how preferences improve the experience)
- [ ] SecurityAlert constraint explained in non-technical user terms
- [ ] Usability requirements are measurable (e.g., "complete in < 30 seconds")
- [ ] WCAG 2.1 AA stated as accessibility baseline
- [ ] `ux-spec` contract produced with all required fields

---

## Fail Criteria

- No personas or only 1 persona → ❌ insufficient user research
- No current journey pain points identified → ❌ missing problem analysis
- SecurityAlert constraint omitted → ❌ missing domain requirement
- Usability requirements are vague ("should be easy") → ❌ not measurable
- No accessibility standard mentioned → ❌ missing a11y commitment
