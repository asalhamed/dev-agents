# Eval: product-owner — 002 — Feature Prioritization

**Tags:** RICE, prioritization, capacity planning, trade-offs
**Skill version tested:** initial

---

## Input (task brief)

```
Prioritize: (1) dark mode, (2) bulk order export, (3) SSO login, (4) mobile app,
(5) notification preferences. 2 engineers, 3 months.
```

---

## Expected Behavior

The product-owner should:
1. Apply RICE scoring or equivalent structured framework
2. Provide rationale for each feature's score
3. Apply the capacity constraint (2 engineers, 3 months)
4. Produce a recommended priority order
5. Explicitly identify features that won't fit this cycle

---

## Pass Criteria

- [ ] Structured prioritization framework used (RICE or equivalent)
- [ ] Each feature scored with visible rationale
- [ ] Capacity constraint applied (2 engineers × 3 months = 6 person-months)
- [ ] Recommended order with clear explanation
- [ ] Won't-do this cycle explicitly listed with reasoning
- [ ] Trade-offs between features acknowledged

---

## Fail Criteria

- No framework — just gut feeling ordering → ❌ not defensible
- Ignores capacity constraint (fits all 5 features) → ❌ unrealistic
- No rationale for scores → ❌ not transparent
- Missing Won't-do list → ❌ pretends everything fits
