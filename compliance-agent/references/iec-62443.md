# IEC 62443 — Industrial Control Systems Security

**Scope:** Mapping our system to the IEC 62443 series for industrial / utility / OT
customers. The standard is mandatory for many water, energy, manufacturing, and
transport deployments.

- **62443-4-2** — technical security requirements for components

## Security Levels (SL-T) (TODO — fill in Phase 3)
- **SL 1** — casual or coincidental violation
- **SL 2** — intentional violation using simple means
- **SL 3** — intentional violation using sophisticated means
- **SL 4** — intentional violation using sophisticated means with extended resources

Per zone and conduit, document the target SL-T and the achieved SL.

## Zones and conduits (TODO — fill in Phase 3)
- **Zone** — group of assets with common security requirements (e.g., "camera fleet at
  site X", "central video management cloud")
- **Conduit** — communication path between zones (e.g., "edge node → cloud ingest")
- Diagram our system as zones + conduits; assign SL-T to each

## System requirements (IEC 62443-3-3) (TODO — fill in Phase 3)
Foundational requirements (FR) we must meet:
1. Identification and authentication control (IAC)
2. Use control (UC)
3. System integrity (SI) — includes secure update (→ `firmware-ota-agent`)
4. Data confidentiality (DC)
5. Restricted data flow (RDF)
6. Timely response to events (TRE)
7. Resource availability (RA)

For each FR × SL-T cell, document control implementation + evidence.

## Component requirements (IEC 62443-4-2) (TODO — fill in Phase 3)
Our cameras, sensors, gateways, and edge-media nodes each need:
- Unique identity (coordinate with `iot-dev` → `hardware-attestation.md`)
- Secure authentication
- Integrity protection
- Secure session management
- Configuration management
- Auditable event records
